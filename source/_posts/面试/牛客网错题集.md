### 1.1.3 如何解决事务的脏读、不可重复读以及幻读？

> + **脏读**：当一个事务**读取到另一个事务尚未提交的修改时**，产生脏读
> + **不可重复读**：同一查询在同一事务中多次进行，由于**其他提交事务所做的==修改或删除==，每次返回不同的结果集**，此时发生不可重复读
> + **幻读**：同一查询在同一事务中多次发生，由于**其他提交事务所做的==插入==操作，每次返回不同的结果集**，此时发生幻读

脏读解决方案：修改时加排他锁，直到事务提交后释放，读取时加共享锁

不可重复读/幻读解决方案：读取数据时加共享锁，写数据时加排他锁，都是事务提交才释放锁

> + **排他锁**允许一个线程或进程独占地对资源进行读写访问，而阻止其他线程或进程对该资源的访问
>   1. **独占性**：排他锁是独占性的，即只有一个线程或进程能够持有排他锁，允许它对资源进行读写操作，其他线程或进程在该锁被持有期间无法访问资源
>   2. **防止并发写入冲突**：排他锁的主要作用是防止并发写入冲突，确保在资源被一个线程或进程写入时不会同时被其他线程或进程写入，从而维护数据的一致性
>   3. **阻塞其他访问**：当一个线程或进程持有排他锁时，其他线程或进程在试图获取相同资源的排他锁时通常会被阻塞，直到当前线程或进程释放了该锁
> + **共享锁**是一种锁定机制，用于控制对共享资源的并发访问
>   1. **非排他性**：共享锁是非排他性的，允许多个线程或进程同时持有共享锁，但不允许对资源进行写入操作
>   2. **避免写入冲突**：共享锁通过阻止对资源的写入访问来避免写入冲突，从而确保在共享资源被读取时不会发生写入操作

### 1.1.4 drop，delete、truncate

```sql
# 删除表
drop/truncate/delete table 表名
```

+ 处理效率

  > drop > truncate > delete

+ 删除范围

  > drop 删除整个表（结构和数据一起删除）；truncate 删除全部记录，但不删除表结构；delete 只删除数据

+ 高水位线

  > delete 不影响自增 ID 值，高水线保持原位置不动；truncate 会将高水线复位，自增 ID 变为 1

### 1.2.1 输出题目真实通过率

牛客刷题记录表`done_questions_record`，为验证重复刷题率，需要我们查找一些数据： 

+ question_pass_rate：表示每个用户不同题目的通过率（同一用户同一题重复提交通过仅计算一次）
+ pass_rate：表示每个用户的提交正确率（只要有提交一次即计算一次）
+ question_per_cnt：表示平均每道不同的题目被提交的次数（只要有一次提交即计算一次）

请你输出题目通过率 question_pass_rate > 60% 的用户的提交正确率 pass_rate 与每题目平均提交次数  question_per_cnt，按照用户名升序排序。result_info '是否通过，1：通过； 0：不通过'

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202403280801680.png" alt="image-20240328080101606" style="zoom:50%;" />

查询返回结果名称和顺序为：

```sql
user_id|question_pass_rate|pass_rate|question_per_cnt
```

----

$$
question\_pass\_rate = \dfrac{diff\_question\_pass\_num}{all\_diff\_question\_num}\\\\
pass\_rate=\dfrac{all\_question\_pass\_num}{all\_question\_num}\\\\
question\_per\_cnt=\dfrac{all\_question\_num}{all\_diff\_question\_num}
$$

+ diff_question_pass_num：每个用户提交不同题目通过总数
+ all_diff_question_num：每个用户提交不同题目总数
+ all_question_pass：每个用户所有提交题目通过次数
+ all_question_num：每个用户提交题目次数

三项指标皆以“每个用户”开头，因此对 user_id 进行分组，接着按照公式对各项信息进行处理即可

> select 中对列起的别名可以在 having 中直接使用

```sql
# question_pass_rate：每个用户不同题目通过率
# pass_rate：每个用户的提交正确率
# question_per_cnt：每个用户平均每道不同的题目被提交的次数
select user_id,
count(distinct if(result_info=1,question_id,null))/count(distinct question_id) as  question_pass_rate,
sum(result_info)/count(result_info) as pass_rate, 
count(result_info)/count(distinct question_id) as question_per_cnt
from done_questions_record 
group by user_id
# result_info=1且question_id不重复的行数
having question_pass_rate > 0.6
order by user_id
```

### 1.2.2 统计复旦用户8月练题情况

现在运营想要了解复旦大学的每个用户在 8 月份练习的总题目数和回答正确的题目数情况，请取出相应明细数据，对于在8 月份没有练习过的用户，答题数结果返回 0

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202403280904258.png" alt="image-20240328090411188" style="zoom:67%;" />

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202403280904233.png" alt="image-20240328090428191" style="zoom: 67%;" />

查询返回结果名称和顺序为：

```sql
device_id|university|question_cnt|right_question_cnt
```

----

+ 已知题目有两个限制条件：①复旦②8月份，一者在 user_profile 表，另一者在 question_practice_detail 表中，可用 where 进行过滤
+ 再通过共有关键字 device_id 对user_profile 表与 question_practice_detail 表进行左连接，此时右表留存的记录即“复旦大学 8 月份有练习的学生”记录
+ 又因为左表数据被完全保留（即满足条件①，无论是否满足条件②），因此按照 device_id 进行分组并通过 count 进行练习数统计后，可以得到所有复旦学生在 8 月的练习总数，没有练习的置 0 

```sql
# 复旦大学的每个用户在8月份练习的总题目数和回答正确的题目数情况
# 在8月份没有练习过的用户，答题数结果返回0
select a.device_id,university,count(question_id) as question_cnt,
sum(case when result = 'right' then 1 else 0 END) as right_question_cnt 
from (select device_id,university 
from user_profile where university = '复旦大学' )a 
# 左连接，a表被全部保留，b表需要满足on条件才被保留
left join (select * from question_practice_detail 
where month(date)='08')b
on a.device_id = b.device_id
group by device_id
```

### 1.2.3 计算用户的平均次日留存率

现在运营想要查看用户在某天刷题后第二天还会再来刷题的平均概率。请你取出相应数据

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202403280933664.png" alt="image-20240328093324624" style="zoom: 67%;" />

----

+ 题目中有两个条件：①有刷题②第二天有刷题。只要出现在 question_practice_detail 表中的数据皆满足条件①，通过左连接（自己连接自己），并使用 DATEDIFF 日期函数即可使右表留存下的数据满足条件①与②
+ 同时，考虑到一个用户在同一天多次刷题只算作一次，因此需要配合 distinct 去重，只统计 (device_id,date) 不重复组合的数量，在左表中这个数量即用户在不同日期的次数，在右表中这个数量即用户在某天刷题后第二天依然刷题的次数

```sql
# count统计日期和设备id的不重复组合
# Count(DISTINCT q1.device_id,q1.date)得到用户在不同日期的刷题数
SELECT COUNT(distinct q2.device_id,q2.date)/count(DISTINCT q1.device_id,q1.date) as avg_ret 
# 左连接主体为q1被全部保存，q2只有符合条件的行才被保存，其余置NULL
# q2 得到相同用户与q1天数差值为1的记录
from question_practice_detail as q1 left join question_practice_detail as q2
on q1.device_id=q2.device_id and DATEDIFF(q2.date,q1.date)=1
```

### 1.2.4 查询每天刷题通过数最多的前二名用户id和刷题数

现有牛客刷题表 questions_pass_record，请查询每天刷题通过数最多的前二名用户 id 和刷题数，输出按照日期升序排序

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202403281020631.png" alt="image-20240328102042586" style="zoom:67%;" />

查询返回结果名称和顺序为：

```sql
date|user_id|pass_count
```

---

+ 题目中隐含三个条件：①每天②用户③前二，按照date，user_id分组可满足条件①②，同时使用 sum 对 pass_count 进行求和即可得到每个用户每天的刷题总数，需要想办法取出前两条数据
+ 此时可以考虑使用 row_number() over() 窗口函数，其能够按照分组、排序规则给每行数据进行编号，接着通过 where 过滤即可得到想要的数据

```sql
select date,user_id,cnt as pass_count
from (select *,
            # 此处仅仅起到编号的作用，不改变实际表 
            row_number() over (
                # 以组日期作为单位，按照刷题量排序
                partition by date order by cnt desc) rw
        # 根据日期和id进行分组，得到同一用户在同一天的刷题量
        from(select user_id,date,sum(pass_count) cnt
                from questions_pass_record
                group by date,user_id) a
      )b
# 按照日期的分组中取出前两行数据     
where rw <= 2 order by date
```

### 1.2.6 平均活跃天数和月活人数

用户在牛客试卷作答区作答记录存储在表exam_record中，内容如下：

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404082212145.png" alt="image-20240408221225052" style="zoom:67%;" />

请计算 2021 年每个月里试卷作答区用户平均月活跃天数 avg_active_days 和月度活跃人数 mau

> 此处活跃指有交卷行为

查询返回结果和顺序为：

```sql
month|avg_active_days|mau
```

---

+ 第一层筛选为 2021 年（ where 过滤）；第二个条件为每个月（分组处理）；第三个信息为活跃人数，即有交卷行为的用户才是统计对象
+ 此题有一个小坑，即一个用户在同一天多次提交，活跃天数算作 1 ，因此使用 count 计数时需要配合 uid 与 submit_time 中的年月日信息进行计数，避免多算

```sql
# 2021年每个月里试卷作答区用户平均月活跃天数avg_active_days和月度活跃人数mau
# 活跃指有交卷行为
select date_format(submit_time,"%Y%m") as month,
# 同一天多次登录只算活跃一次，round保留两位小数
round(count(distinct uid,date_format(submit_time,"%Y%m%d"))/count(distinct uid),2) as avg_active_days,
count(distinct uid) as mau
from exam_record
where year(submit_time) ='2021'
group by month
```

### 1.2.7 未完成试卷数大于1的有效用户

现有试卷作答记录表 exam_record

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404091058913.png" alt="image-20240409105758803" style="zoom:67%;" />

还有一张试卷信息表 examination_info

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404091100588.png" alt="image-20240409110035538" style="zoom:67%;" />

请统计 2021 年每个未完成试卷作答数大于 1 的有效用户的数据（有效用户指完成试卷作答数至少为 1 且未完成数小于5 ），输出用户ID、未完成试卷作答数、完成试卷作答数、作答过的试卷 tag 集合

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404091101468.png" alt="image-20240409110151429" style="zoom:67%;" />

---

+ 第一层筛选为 2021 年（where）；第二层筛选，以用户作为最小单位（group）；第三层筛选，分组后试卷作答数大于 1，完成试卷作答数至少为 1，未完成数小于 5 的用户（having）
+ detail 由两表连接后同一组的 `start_time:tag` 按行拼接组成，聚合后跨行的拼接需要使用 group_concat 函数，行内列拼接则需要使用 concat_ws 函数

```sql
# 统计2021年每个未完成试卷作答数大于1的有效用户的数据
#（有效用户指完成试卷作答数至少为1且未完成数小于5）
select uid,sum(if(submit_time is null,1,0)) as incomplete_cnt,
sum(if(submit_time is null,0,1)) as complete_cnt,
group_concat(distinct concat_ws(':',date(start_time),tag) order by start_time separator ';') as detail
from exam_record left join examination_info using(exam_id)
WHERE year(start_time) = 2021
group by uid
having incomplete_cnt>1 and complete_cnt>=1 and incomplete_cnt<5
order by incomplete_cnt	desc
```

### 1.2.8 分别满足两个活动的人

假使以前有两拨运营活动，分别给每次试卷得分都能到 85 分的人（activity1）、至少有一次用了一半时间就完成高难度试卷且分数大于80的人（activity2）发了福利券

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404111108925.png" alt="image-20240411110825783" style="zoom:80%;" />

请写出一个 SQL 实现：输出 2021 年里，所有每次试卷得分都能到 85 分的人以及至少有一次用了一半时间就完成高难度试卷且分数大于 80 的人的 id 和活动号，按用户 ID 排序输出

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404111108507.png" alt="image-20240411110855394" style="zoom:67%;" />

---

+ 从结果中可以看到，同时满足两个条件的用户会被输出两次，因此大概率会使用 union all 连接两个条件筛选出的用户

+ 条件一：2021年，每次分数大于等于 85

  条件二：2021年，高难度试卷分数有大于 80 的情况，考试中至少一次用了一半时间完成

+ timestampdiff(unit,start_time,submit_time) 的使用，得到两个时间指定单位的时间差，此处需要用秒，因为需要考虑超出不足一分钟的情况，此时 minute 单位是相同的

```sql
select uid,"activity1" as activity
from exam_record
where year(start_time)=2021
group by uid
having min(score) >= 85
UNION ALL
select uid,"activity2" as activity
from exam_record
left join examination_info using(exam_id)
where year(start_time)=2021
and difficulty="hard"
and score>80
and TIMESTAMPDIFF(SECOND,start_time, submit_time) <= duration*30
# 写在最后是整体排序
order by uid;
```

找到高难度 SQL 试卷得分平均值大于 80 并且是 7 级的红名大佬，统计他们的 2021 年试卷总完成次数和题目总练习次数，只保留 2021 年有试卷完成记录的用户。结果按试卷完成数升序，按题目练习数降序

<img src="https://z-cloud-pic-1313046262.cos.ap-guangzhou.myqcloud.com/img/202404111224903.png" alt="image-20240411122459795" style="zoom: 80%;" />

---

+ 首先统计的对象为"高难度 SQL 试卷得分平均值大于 80 并且是 7 级的红名大佬"，此处可以过滤得到目标 uid ，处理后，user_info 表在此题中不再需要

+ 第二个点是 2021 的试卷总完成数和题目总练习数。首先完善关于将过滤条件写在连接条件中与 where 中的区别。 

  当在连接条件中完成过滤，此时记录被过滤但是表行内容并不减少，仅被置空；如果通过 uid 完成左连接后再进行过滤，则行数据会被抹去，以下是一个小例子说明二者的不同之处

  ```sql
  table1
  +------+--------+
  | uid  |  value |
  +------+--------+
  |  1   |   A    |
  |  2   |   B    |
  |  3   |   C    |
  +------+--------+
  
  table2
  +------+--------+
  | uid  |  value |
  +------+--------+
  |  1   |   X    |
  |  3   |   Y    |
  +------+--------+
  ```

  1. 将条件放在`LEFT JOIN`的`ON`子句中：

     ```sql
     SELECT *
     FROM table1 t1
     LEFT JOIN table2 t2 ON t1.uid = t2.uid AND t2.value = 'X';
     ```

     结果是：

     ```sql
     +------+--------+------+--------+
     | t1.uid | t1.value | t2.uid | t2.value |
     +------+--------+------+--------+
     |   1   |    A     |   1    |    X     |
     |   2   |    B     |  NULL  |  NULL    |
     |   3   |    C     |  NULL  |  NULL    |
     +------+--------+------+--------+
     ```

  2. 将条件放在`WHERE`子句中：

     ```sql
     SELECT *
     FROM table1 t1
     LEFT JOIN table2 t2 ON t1.uid = t2.uid
     WHERE t2.value = 'X';
     ```

     结果是：

     ```sql
     +------+--------+------+--------+
     | t1.uid | t1.value | t2.uid | t2.value |
     +------+--------+------+--------+
     |   1   |    A     |   1    |    X     |
     +------+--------+------+--------+
     ```

  2021年完成试卷的人中，可能没有练习过；2021年完成练习的人中，可能没有参加考试。因此使用 where 过滤可能导致 count 计数发生错误，所以需要在连接时过滤数据

+ 另一个小坑为：试卷总完成次数指不同试卷的完成次数，题目总练习数指所有题目完成次数，但是由于经过左连接操作，因此在统计时需要使用 distinct 进行去重

```sql
# 高难度SQL试卷得分平均值大于80并且是7级的红名大佬
# 统计他们的2021年试卷总完成次数和题目总练习次数，只保留2021年有试卷完成记录的用户
```

---

+ 首先，归一化公式中的最大最小值面对的对象是同一类型难度为 hard 的试卷；而归一化后分数平均值是相对用户而言，分组依据是 uid,exam_id
+ 没有得分的数据不纳入归一化统计，只有一条数据的试卷不套用归一化公式

```sql
#高难度试卷的得分在每份试卷作答记录内执行min-max归一化后缩放到[0,100]区间，并输出用户ID、试卷ID、归一化后分数平均值
select uid,
    exam_id,
    # 求所有归一化数据求平均值
    round(avg(min_max),0) as avg_new_score
from
    # 没有分数不进行归一化，只有一行数据不需要归一化
    (select *,
        if(max_score=min_score,score,(score-min_score)/(max_score-min_score)*100) as min_max
    from 
        # 得到hard试卷各类试卷的最大最小分数
        (select uid,a.exam_id,score,
            max(score) over(partition by a.exam_id) as max_score,
            min(score) over(partition by a.exam_id) as min_score
        from exam_record a 
        left join examination_info b using(exam_id)
        where difficulty='hard'
    ) t
    where score is not null) t1
group by uid,exam_id
order by exam_id asc,avg_new_score desc;
```

# 4. 测试

## 4.1 概念相关

### 4.1.1 逻辑测试覆盖中，覆盖程度由弱到强顺序是什么？

> 覆盖是测试人员之间常用的交流语言，也即测试到的地方称为覆盖

1. 语句覆盖 ：可执行语句至少被执行一次

   > 每行代码都被执行

2. 判定覆盖：每个判断的取真/取假分支至少经历一次

   > 关注分支结构（if...else等）的 True 和 False 分支是否能被正确覆盖

3. 条件覆盖：每个条件取值至少满足一次

   > 在整个测试过程中，逻辑判断子条件取值（True 或 False）至少一次，但是不关心取值的排列组合

   ```java
   public void foo (int a, int b, int x) {
       if(a>1 && b==0) {
           x = x/a;
       }
   	 
       if (a==2 || x>1) {
           x = x+1;
       }
   }
   ```

   第一个判断的所有条件的可能取值情况是 a>1 或 a≤1 ，b=0 或 b≠0 ；第二个判断条件可能的取值情况为 a=2 或 a≠2 ，x>1 或 x≤1

   ```java
   // 以下测试用例即可满足所有条件覆盖（所有条件值都被取到）
   a=1，b=0，x=3 
   a=2，b=1，x=1 
   ```

   然而在这种情况下第二个判断条件却无法取值为 False，因此满足条件覆盖不一定满足判定覆盖

4. 判定/条件覆盖：判定覆盖和条件覆盖都要满足

5. 条件组合覆盖：每个条件的所有可能都至少出现一次，并且判定结果至少出现一次

   > 关注关注分支结构（if...else等）中所有单个子条件的 True 和 False 组合情况是否能被正确覆盖

   ```java
   public static void main(String[] args) {
   		Scanner sc=new Scanner(System.in);
   		int x=sc.nextInt();
   		int y=sc.nextInt();
   		int num=sc.nextInt();
   		if(x>0 && y>0) {
   			System.out.println("Hello1");
   		}else{
   			System.out.println("Hello2");
   		}
   		if(num < 0) {
   			System.out.println("Hello3");
   		}
   		System.out.println("Hello4");
   	}
   ```

   + 第一个 if 语句（x>0 && y>0）定义为 test1
   + 第二个 if 语句（num < 0）定义为 test2
   + x>0 定义为 p1；y>0 定义为 p2；num<0 定义为 p3

   以下测试用例满足条件覆盖

   | 测试数据           | p1   | p2   | p3   | test1 | test2 |
   | ------------------ | ---- | ---- | ---- | ----- | ----- |
   | {x=1,y=1,num=2}    | T    | T    | F    | T     | F     |
   | {x=-1,y=-1,num=-2} | F    | F    | T    | F     | T     |
   | {x=1,y=-1,num=2}   | T    | F    | F    | F     | F     |
   | {x=-1,y=1,num=-2}  | F    | T    | T    | F     | T     |

   满足条件组合覆盖可能不会覆盖所有的路径，如 test1=True 与 test2=True 的情况此处未被覆盖

6. 路径覆盖：执行所有可能的执行路径

   > 将程序执行抽象为流程图后所有路径进行覆盖

### 4.1.2 各类测试分别对应什么文档？

**v 模型**

用户需求<————————————————>验收测试

​       需求分析<————————————>系统测试

​              概要设计<————————>集成测试

​                     详细设计<————>单元测试

​                                       	编码

+ **单元测试**：详细设计文档 

  > 单元测试，又称模块测试。对软件的组成单位进行测试，其目的是检验软件**基本组成单位**的正确性。
  > 测试的对象的是软件测试的最小单位：模块

+ **集成测试**：概要设计文档

  > 集成测试也称联合测试（联调）、组装测试，将程序模块采用适当的集成策略组装起来，对系统的接口及**集成后的功能**进行正确性检测的

+ **系统测试**：需求分析设计文档

  > 系统测试将软件系统看成是一个**系统的测试**。包括对功能、性能以及软件所运行的软硬件环境进行测试。
  > 测试所花费的时间大部分在系统测试执行阶段，其从完整的角度，广面去看待问题，不再看模块，包括回归测试和冒烟测试（**先冒烟、再系统、后回归**）

### BM12 单链表的排序

> 给定一个节点数为n的无序单链表，对其按升序排序。

要求：空间复杂度 O(n)，时间复杂度 O(nlogn)

---

+ 时间复杂度为 O(nlogn) 的排序算法为快速排序、归并排序、堆排序。快速排序与堆排序依靠数组索引反复跳变取值，在链表中不具备可操作性，因此此题实则要求对单链表进行**归并排序**
+ 归并排序的思想是先按中间位置反复切割至最小单位，再比较各部分大小情况，逐渐拼接为完整数据，在数组中可以很容易得到 mid 位置，而在链表中则需通过快慢指针来完成 mid 的定位，同时还需要使用 mid 的前置指针 pre，以便将完整的链表一分为二
+ merge 操作则是将两个链表升序连接，分别对两个链表遍历，用头结点按照大小顺序连接即可

```java
public class Solution {
    public ListNode sortInList (ListNode head) {
        if(head==null || head.next==null){
            return head;
        }
        return mergeSort(head);
    }
    public ListNode mergeSort(ListNode head){
        if(head.next==null){
            return head;
        }
        ListNode low=head,fast=head,pre=null;
        while(fast!=null && fast.next!=null){
            pre=low;
            low=low.next;
            fast=fast.next.next;
        }
        // 从中间截断链表
        pre.next=null;
        ListNode left = mergeSort(head);
        ListNode right = mergeSort(low);
        return merge(left,right);
    }
    public ListNode merge(ListNode left,ListNode right){
        ListNode head = new ListNode(-1);
        ListNode cur = head;
        while(left!=null && right!=null){
            if(left.val<right.val){
                cur.next = left;
                left = left.next;
            }else{
                cur.next = right;
                right = right.next;
            }
            cur=cur.next;
        }
        if(left!=null){
            cur.next = left;
        }else{
            cur.next = right;
        }
        return head.next;
    }
}
```

### BM20 数组中的逆序对

> 在数组中的两个数字，如果前面一个数字大于后面的数字，则这两个数字组成一个逆序对。输入一个数组，求出这个数组中的逆序对的总数 P 。并将 P 对 1000000007 取模的结果输出。 即输出 P mod 1000000007

要求：空间复杂度 O(n)，时间复杂度 O(nlogn)

```python
输入：[1,2,3,4,5,6,7,0]
返回值：7
```

---

+ 本题最容易想到的思路是冒泡升序排序，交换次数即为逆序对数量，但是此时时间复杂度为O(n²)，不符合题目要求
+ 另一个思路是**归并升序排序**，每次 merge 的 [left,mid]，[mid+1,right] 内部升序，当后一部分第 j 个元素小于前一部分第 i 个元素时，说明出现了逆序对，并且由于内部有序，因此元素 j 实际上小于 [i,mid] 之间的所有元素，即逆序对数量为 mid-i+1
+ 使用全局变量 count 对逆序对数量进行统计，最后取模得到答案

```java
public class Solution {
    long count = 0;
    public int InversePairs (int[] nums) {
        if(nums.length<=1){
            return -1;
        }
        // 归并排序
        MergeSort(nums,0,nums.length-1);
        return (int)(count % 1000000007);
    }
    public void MergeSort(int[] nums,int left,int right){
        if(left<right){
            int mid = (left+right)/2;
            MergeSort(nums,left,mid);
            MergeSort(nums,mid+1,right);
            Merge(nums,left,right,mid);
        }
    }
    public void Merge(int[] nums,int left,int right,int mid){
        int temp[]=new int[right-left+1];
        int i =left,j=mid+1,index=0;
        while(i<=mid && j<=right){
            if(nums[i]>nums[j]){
                // 此时mid左右两端各自升序
                // 当if成立时，[i,mid]的元素都大于nums[j]
                count = count + mid - i + 1;
                temp[index++]=nums[j++];
            }else{
                temp[index++]=nums[i++];
            }
        }
        while(i<=mid){
            temp[index++] = nums[i++];
        }
        while(j<=right){
            temp[index++] = nums[j++];
        }
        // 源数组、源数组起始位置、目标数组、目标数组起始位置、复制元素数量
        System.arraycopy(temp,0,nums,left,index);
    }
}
```

### BM21 旋转数组的最小位置

> 有一个长度为 n 的非降序数组，比如 [1,2,3,4,5] ，将它进行旋转，即把一个数组最开始的若干个元素搬到数组的末尾，变成一个旋转数组，比如变成了 [3,4,5,1,2]，或者 [4,5,1,2,3] 这样的。请问，给定这样一个旋转数组，求数组中的最小值。

---

+ 由于原数组为非降序数组，因此旋转之后，子数组内部依旧保持非降序，并且旋转到后面的子数组 2 整体小于前面的子数组 1
+ 基于上述信息，不难发现可以采用二分法思想，一步步迫近子数组 1 的末尾以及子数组 2 的开头
+ 特殊情况为：当 left 指向的元素与 right 指向的元素大小一致时，无法判断 mid 属于子数组 1 还是 2 ，因此不能进一步迫近，此时只能采用顺序遍历寻找最小值

```java
public class Solution {
    public int minNumberInRotateArray (int[] nums) {
        int left=0,right=nums.length-1;
        // 此时没有旋转
        if(nums[left]<nums[right]){
            return nums[left];
        }
        while(nums[left]>=nums[right]){
            if(right-left==1){
                break;
            }
            int mid = (left+right)/2;
            if(nums[left]==nums[right]){
                // 此时无法分辨mid属于哪个子数组，只能顺序遍历
                int temp = nums[left];
                for(int i =left+1;i<right;i++){
                    if(nums[i]<temp){
                        temp=nums[i];
                    }
                }
                return temp;
            }
            if(nums[mid]>=nums[left]){
                left = mid;
            }else{
                right = mid;
            }
        }
        return nums[right];
    }
}
```
